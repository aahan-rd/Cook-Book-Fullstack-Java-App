Why did we choose this feature?
We chose Export to PDF because it directly extends the Import Interface feature built in GA1. The Import flow lets users bring recipes into the app from images while the export to PDF closes the loop by letting users take those recipes back out in a portable, shareable format. It felt like a natural complement to what we already built.

What user needs does it address?
This feature addresses the need of users who want to share recipes outside the app or use them offline. From Willem Austin, a home cook who collects recipes from multiple sources needs a way to consolidate and share them with family members who don't use the app. A printed or emailed PDF is the most universal format for this as it works on any device, requires no account, and can be physically printed for use in the kitchen.

What alternatives did we consider?

Recipe Scaling: We considered scaling recipes by serving size with automatic ingredient recalculation. While useful, it doesn't directly address the sharing use case our persona needed. Furthermore, the way our recipe editor is set up from import, makes it perfect 

Cooking Timer: We considered timers linked to recipe steps. This is higher complexity and while useful in the kitchen, it doesn't address the core need of sharing recipes with others. It also may not set as intuitively without UI as export.
